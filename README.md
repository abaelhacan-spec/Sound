# Sound Monitor App — تطبيق مراقبة صوتية

تطبيق Expo/React Native يراقب صوت منبه أو جهاز إنذار محدد، وعند التعرف عليه (عبر
مطابقة البصمة الترددية للصوت)، يتصل تلقائيًا برقم هاتف محدد مسبقًا.

## ⚠️ ملاحظات مهمة قبل البدء

1. **هذا التطبيق لا يعمل على Expo Go** — لأنه يحتاج مكتبة `react-native-immediate-phone-call`
   للاتصال التلقائي المباشر، وهذه مكتبة native لا تدعمها بيئة Expo Go الجاهزة.
   لازم تعمل **Development Build** أو **Prebuild + بناء APK**.
2. **يعمل على أندرويد فقط** — صلاحية `CALL_PHONE` بهذا الشكل غير مسموحة على iOS إطلاقًا.
3. **صلاحية `CALL_PHONE` حساسة** — أندرويد هيطلب منك تفعيلها يدويًا من إعدادات
   التطبيق بعد التثبيت (Settings > Apps > Sound Monitor > Permissions).
4. الشاشة يجب أن تبقى مفتوحة والتطبيق في المقدمة أثناء المراقبة (حسب طلبك، تم
   تصميمه على أساس ترك الهاتف الثانوي مفتوحًا على التطبيق طوال الوقت).

## 📁 هيكل المشروع

```

├── babel.config.js
├── package.json                # قائمة المكتبات المطلوبة
├── screens/
│   ├── CalibrationScreen.js    # شاشة تسجيل صوت المنبه المرجعي + رقم الهاتف
│   └── MonitoringScreen.js     # شاشة المراقبة الحية والاتصال التلقائي
└── utils/

    └── storage.js              # حفظ/استرجاع الإعدادات محليًا (AsyncStorage)
```

## 🚀 خطوات التشغيل

### 1. تثبيت المتطلبات

```bash
cd monitor-app
npm install
```

### 2. تشغيل Prebuild لتوليد مجلد Android الأصلي

```bash
npx expo prebuild --platform android
```

هذا الأمر بيولّد مجلد `android/` يحتوي على AndroidManifest.xml وكل الكود الأصلي
اللازم. الصلاحيات المطلوبة (RECORD_AUDIO, CALL_PHONE, WAKE_LOCK) متضافة بالفعل
عبر `app.json`.

### 3. التأكد من الصلاحيات في AndroidManifest.xml (تحقق يدوي)

افتح `android/app/src/main/AndroidManifest.xml` وتأكد من وجود:

```xml
<uses-permission android:name="android.permission.RECORD_AUDIO" />
<uses-permission android:name="android.permission.CALL_PHONE" />
<uses-permission android:name="android.permission.WAKE_LOCK" />
```

### 4. بناء وتشغيل التطبيق على جهاز أندرويد حقيقي

وصّل هاتف الأندرويد (المخصص للمراقبة) بالكمبيوتر عبر USB وفعّل وضع المطور +
USB Debugging، ثم:

```bash
npx expo run:android
```

هذا الأمر هيبني APK ويثبته مباشرة على الجهاز الموصول.

**بديل: بناء APK جاهز للتوزيع بدون كمبيوتر متصل دائمًا (عبر EAS Build):**

```bash
npm install -g eas-cli
eas login
eas build:configure
eas build --platform android --profile preview
```

ده هيبني APK على سيرفرات Expo وهيديك رابط تحميل مباشر تقدر تنزله على الهاتف
وتثبته يدويًا (لازم تفعيل "تثبيت من مصادر غير معروفة" في إعدادات أندرويد).

### 5. منح صلاحية الاتصال يدويًا (إجراء لازم مرة واحدة)

بعد التثبيت، اذهب إلى:
`إعدادات الهاتف > التطبيقات > Sound Monitor > الصلاحيات`
وفعّل صلاحية **"الهاتف / Phone"** يدويًا إذا لم تظهر رسالة الطلب تلقائيًا.

## 🎛️ طريقة الاستخدام

1. **افتح التطبيق أول مرة** → هتظهر شاشة الإعداد (Calibration)
2. **أدخل رقم الهاتف** المطلوب الاتصال به عند التنبيه
3. **اضغط "تسجيل عينة"** ثلاث مرات أثناء تشغيل صوت المنبه الحقيقي بجانب الهاتف
   (كل عينة 1.5 ثانية) — كل ما العينات أدق، كل ما التعرف أفضل
4. **اضغط "حفظ وإنهاء الإعداد"** → هينتقل تلقائيًا لشاشة المراقبة
5. **اضغط "ابدأ المراقبة"** → التطبيق هيبدأ يسجل مقاطع قصيرة (1.2 ثانية) بشكل
   متكرر، ويقارن كل مقطع بالبصمة المرجعية
6. لو حصل تطابق 3 مرات متتالية (قابل للتعديل) → هيتصل تلقائيًا بالرقم المحفوظ

## ⚙️ إعدادات قابلة للتعديل (داخل الكود)

في ملف `screens/MonitoringScreen.js`:

```javascript
const CONSECUTIVE_MATCHES_NEEDED = 3;  // عدد المقاطع المتتالية المطابقة قبل الاتصال
const COOLDOWN_MS = 30000;             // مهلة (بالمللي ثانية) قبل السماح باتصال تالٍ
const CHUNK_DURATION_MS = 1200;        // مدة كل مقطع تسجيل يُحلَّل
```

في ملف `utils/storage.js` (أو من شاشة الإعداد لاحقًا لو حبيت تضيف واجهة تعديل):

```javascript
threshold = 0.85  // عتبة التشابه (0 إلى 1) — كل ما زادت كل ما التطابق أدق وأصعب
```

## 🔧 ضبط الدقة (Threshold Tuning)

القيمة الافتراضية لعتبة التشابه هي `0.85`. بعد أول تجربة فعلية:

- **لو مفيش اتصالات بتحصل رغم سماع صوت المنبه** → قلّل العتبة تدريجيًا (مثلاً 0.75)
- **لو بيتصل بالغلط من أصوات تانية** → زوّد العتبة (مثلاً 0.90)

الأفضل تجرب في نفس البيئة الفعلية (نفس الغرفة، نفس المسافة) وتظبط القيمة حسب
النتيجة العملية.

## 🐛 استكشاف الأخطاء الشائعة

| المشكلة | الحل المحتمل |
|---|---|
| التطبيق ميديش صلاحية `CALL_PHONE` | فعّلها يدويًا من إعدادات النظام كما بالخطوة 5 |
| `react-native-immediate-phone-call` مش شغال | تأكد إنك عملت `expo prebuild` ومش شغال على Expo Go |
| دقة التعرف ضعيفة | سجّل عينات معايرة أكتر (عدّل `SAMPLES_NEEDED` في `CalibrationScreen.js`) وجرب في نفس ظروف الاستخدام الفعلية |
| التطبيق بيستهلك بطارية بسرعة | وصّل الهاتف بالشاحن دائمًا أثناء المراقبة (متوقع بسبب التسجيل المستمر + إبقاء الشاشة مضاءة) |
| ملفات WAV مؤقتة بتتراكم | الكود بيمسحها تلقائيًا بعد كل تحليل عبر `deleteTempFile` — تأكد إن الحلقة مش بتتوقف بخطأ قبل الحذف |

## 📝 ملاحظة تقنية عن دقة التعرف

هذا الحل يعتمد على **البصمة الترددية (Spectral Fingerprint)** عبر تحويل فورييه
السريع (FFT)، وهو أدق بكثير من مجرد قياس علو الصوت (dB)، لكنه ليس بدقة نماذج
الذكاء الاصطناعي المتقدمة (مثل YAMNet). في بيئة هادئة مع صوت منبه ثابت (كما هي
حالتك)، من المتوقع أن يعطي دقة عالية جدًا وموثوقة عمليًا.
